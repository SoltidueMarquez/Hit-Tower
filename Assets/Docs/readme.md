已实现清单：

**核心功能**

- 地图/路径：1 条主路径（可拐弯）。怪物使用 **NavMesh/NavMeshAgent** 。
- 敌人与波次：波次 JSON 配置（怪物类型、数量、间隔、出怪口）；
- 精英怪具特殊抗性（对物理伤害减半，也只有物理伤害，通过配置伤害吸收率实现）。
- 塔与战斗：至少 3 类塔（单体高伤、群体减速、持续灼烧 DoT），可升级 2 级。
- Buff 系统：
  - 塔端 Buff（攻速↑、溅射、穿透，后面两个只对单体塔有意义）。
  - 怪端 Buff（狂暴、护甲、易伤）。
  - 冲突/叠层规则（如同类取最高、不同类可叠乘）。
- 经济与商店：击杀得币，建塔/升级花费，商店临时 Buff（限时 30s）。
- 存档：最高波次、玩家解锁（通过了一定波次数目的存档可以解锁更多的塔来战斗）。
- UI：建造面板（悬停显示 DPS 预估）、波次时间轴、倍速控制、暂停/设置。

**技术要求**

- **数据驱动**：塔/怪/Buff/波次皆可由SO配置驱动。
- **编辑器工具（至少 1 个）**：
  - 自定义 Inspector 或小工具窗口：可一键导入/导出波次 JSON；
- **性能/调试**：
  - 在 200 个敌人同屏下保持稳定（使用了Computer Shader优化）。
  - 提供一个简单的 Debug Overlay（FPS、存活数、活跃 Buff 数）。

- 核心玩法：塔与敌人克制明显（减速克制精英怪的狂暴，怪物护甲克制灼烧塔的高频低伤害）。
- 系统深度：Buff 叠层与冲突可验证、波次配置生效。
- 工程质量：数据驱动与编辑器工具可用；代码规范（大概）。
- 性能与稳定：（200怪）压力场景可跑。
- UI 与可用性：信息清晰、操作直观。

**加分项**

- Addressables 动态加载塔/怪资源。  （没做）
- 简易回放（记录关键指令与随机种子）。（没做）
- **仓库结构**：`Assets/Scripts`, `Assets/Art`, `Assets/Scenes`, `Assets/Resources` 或 Addressables；`Docs/` 放技术说明。
